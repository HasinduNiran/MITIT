## Project Overview

Both implementations deliver the same feature set:

- User **registration**, **login**, and **profile** viewing
- **Node.js / Express** backend with **MongoDB** (Mongoose)
- **JWT** authentication + **bcrypt** password hashing
- **Joi** input validation, **Helmet** security headers, **rate limiting**
- **React 18** frontend with **Tailwind CSS**, **React Router v6**, and **Axios**
- Protected routes, toast notifications, and responsive design

## Quick Start

Each implementation runs independently. Pick one and follow its README.

### Prerequisites

- **Node.js 18+** — https://nodejs.org
- **MongoDB** locally on port `27017`, or a [MongoDB Atlas](https://cloud.mongodb.com) connection string

### ChatGPT Implementation

```bash
# Backend
cd Chatgpt/backend
copy .env.sample .env      # fill in MONGO_URI and JWT_SECRET
npm install
npm run dev                # http://localhost:5000

# Frontend (new terminal)
cd Chatgpt/frontend
copy .env.sample .env
npm install
npm run dev                # http://localhost:5173
```

### Claude Implementation

```bash
# Backend
cd Claude/backend
copy .env.example .env     # fill in MONGO_URI and JWT_SECRET
npm install
npm run dev                # http://localhost:5000

# Frontend (new terminal)
cd Claude/frontend
copy .env.example .env
npm install
npm run dev                # http://localhost:5173
```

## Key Differences at a Glance

| Aspect | ChatGPT | Claude |
|--------|---------|--------|
| **bcrypt library** | `bcrypt` (native C++) | `bcryptjs` (pure JS) |
| **Password hashing** | Manual in controller | Pre-save hook in model |
| **JWT claims** | `sub`, `email`, `name`, `iss`, `aud` | `id` only |
| **Rate limiting** | All auth routes | Register + login only |
| **Auth persistence** | Token only; profile fetched from API on refresh | Token + user JSON in localStorage |
| **Request logging** | Morgan | None |
| **Vite dev proxy** | Not configured | Configured for `/api` |
| **Mobile menu** | No | Hamburger menu |
| **Dashboard** | Simple profile card | Gradient banner, avatar, security status |
| **Loading skeleton** | No | Yes |
| **Remember Me** | Yes | No |
| **404 page** | Redirect to `/` | Custom page |

For the full comparison see [Documentation.md](./Documentation.md).

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|

## Security Features (both)

- Passwords hashed with **bcrypt / bcryptjs**, 12 salt rounds
- **JWT** with 1-hour expiry
- **Joi** validation on all inputs
- **5 requests / 15 min** rate limiting on auth endpoints
- **Helmet** security headers
- **CORS** restricted to frontend origin
- JSON body size limited to **10 KB**
- Generic `"Invalid email or password"` prevents user enumeration
